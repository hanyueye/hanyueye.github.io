Iterator 与 ListIterator 的区别

问：既然其父类已经有了迭代器，那么为什么还要创建一个特定的迭代器呢？

因为其父类的迭代器，在迭代时，不可以通过集合对象的方法对集合中的元素进行操作，不然会发生 ConcurrentModificationException 异常。

因此，在迭代时，只能用迭代器的方法操作元素，可是 Iterator 的方法是有限的，只有 hasNext、next、remove 三个方法，其中对元素操作的只有一个 remove 方法。

所以，就出现了 List 集合特有的迭代器，用来在迭代过程中对 List 集合中的数据进行增删该查功能。

问：ListIterator 中有哪些操作元素的方法

问：为什么只有 List 集合中可以使用特有迭代器在迭代过程中对集合进行操作

因为 List 集合中的元素使有序的，有索引（角标）的，所以可以在迭代过程中进行增删该查操作。

List 常用子类对象

问：List 集合根据底层实现的数据结构的不同，又细分了三个子类：

  • Vector：JDK 1.0 出现的，底层使用的数据结构是数组。特点：增删改查速度都不快，因为是线程同步。
  • ArrayList：底层使用的数据结构也是数组。特点：所以查询速度很快，但是增删稍慢。线程不同步，
  • LinkedList：LinkedList 底层使用的数据结构是链表。特点：增删速度快，查询较慢。

问：Vector 和 ArrayList 底层都是用数组实现的，数组是不可变长的，那么为什么在 List 中是可变长呢？

Vector 的初始长度是 10，当数据个数超过 10 时，就创建一个新的数组，将现有的数据 Copy 到新数组中，并将元素加到新数组之后，每次扩容新数组的长度为，当前长度的一倍。

ArrayList 的初始长度也是 10，但是每次扩容的长度为当前长度的 50%。
