Set 特点

  • Set接口中不能加入重复元素，无序
  • 取元素只能通过迭代器，不保证迭代顺序。
  • 接口中的方法与 Collection 一致。

Set 常见方法

由于 Set 接口的所有方法都与 Collection 一致，就不在此再次列举，请到 Java基础：集合框架之Collection 中查看。

Set 常用子类对象

HashSet

内部数据结构是哈希表，线程不同步。

问：如何保证该集合的元素唯一性呢？

  • 是通过对象的 hashCode 和 equals 方法来完成对象唯一性的。
  • 如果对象的 hashCode 值不同，那么不用判断 equals 方法，就直接存储到哈希表中。
  • 如果对象的 hashCode 值相同，那么要再次判断对象的 equals 方法是否为 true。如果为true，视为相同元素，不存。如果为false，那么视为不同元素，就进行存储。

既然存储时是用 hashCode 和 equals 方法来确定对象唯一性的，那么在判断和删除时，也肯定是会先调用这两个方法来确定是否是某个元素，再进行操作。

重点：如果元素要存储到HashSet集合中，必须覆盖hashCode方法和equals方法。

  • 一般情况下，如果定义的类会产生很多对象，比如人，学生，书，通常都需要覆盖equals，hashCode方法（建立对象判断是否相同的依据）。

TreeSet

内部数据结构是红黑树，可以对数据进行排序，线程不同步。
红黑树：红黑树本质上是一棵一定程度上相对平衡的二叉搜索树。

特点：可以对集合中的元素进行排序

因为可以对数据进行排序，所以 TreeSet 要求存入的数据必须具有有比较性。
对于基本数据类型，其对于的基本数据类型包装类都有默认的比较方式。
而对于引用数据类型，就需要自定义数据的比较性了。

问：如何让元素具备比较性？

使元素具备比较性有两种方式：

  1. 使元素实现 Comparable 接口，重写 compareTo 方法，根据其方法的返回值，确定元素的大小。
  2. 创建一个类来实现 Comparator 接口，重写 compare 方法，在 TreeSet 创建时，通过 TreeSet 的构造方法，将比较器当做集合中元素进行比较的方式。

示例 1：使元素实现 Comparable 接口来具备可比较性。

输出结果：

bb..18
bc..10

示例 2：创建 Comparator 比较器来指定元素的可比较性

运行结果：

bc..10
bb..18

根据上方的两个例子，可以得知：

  • 当元素有比较方法，同时 TreeSet 也有比较器时，会优先使用比较器进行比较。
  • 当元素有多个属性时，要先根据其主要属性进行比较，再进行次要元素的比较，如若只对其主要属性进行比较，那么当主要属性相同时，就视为同一元素，那么就无法存入 Set 集合了。
  • 使用比较器更优于使元素本身具备比较性，因为那个元素不一定有权进行修改其源码，使用比较器更为灵活。
