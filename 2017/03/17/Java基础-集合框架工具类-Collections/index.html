Java基础：集合框架工具类之Collections

发表于 2017-03-17 | 分类于 Java基础 |

简介

Collections 是个 Java.util 下的类，是针对集合类的一个工具类，提供一系列静态方法，实现对集合的查找、排序、替换、线程安全化（将非同步的集合转换成同步的）等操作。

常用方法

sort

因为 Map 集合中有 TreeMap，Set 集合有 TreeSet，List 集合没有可排序的数据类型，所以提供了这个方法。

方法定义：

看起来可以比较复杂，其实就是传入一个 List 集合的元素，List 集合中的元素必须具备比较性，才能对元素进行排序。

第一个方法是指元素本身必须实现 Comparable 接口。
第二个方法是指传入一个比较器。

max

此方法就是取 Collection 中最大的元素，其实也就是 sort 排序后，最后一个元素。既然求最大，那么元素也一定要具备可比较性才能求，所以其方法定义上也限制了元素必须实现 Comparable 接口，或者传入一个比较器。

方法定义：

binarySearch

使用二分搜索法搜索指定列表，以获得指定对象。在进行此调用之前，必须根据列表元素的自然顺序对列表进行升序排序（通过 sort(List) 方法）。如果没有对列表进行排序，则结果是不确定的。如果列表包含多个等于指定对象的元素，则无法保证找到的是哪一个。

方法定义：

如果搜索键包含在列表中，则返回搜索键的索引；否则返回 (-(插入点) - 1)

例如：在集合{"aaa","bbb","ccc"} 中查找 "abc"，插入点应该是 "aaa" 后面，"bbb" 之前那个位置，也就是 1，插入点取反再减 1， 也就是 2

replaceAll

就是使用另一个值替换列表中所有的某一指定值

方法定义：

对于集合 list = {"a","a","b","c"}，使用方法replaceAll(list), "a", "d" 后，集合将变为 {"d","d","b","c"}

reverse

反转指定列表中元素的顺序。

方法定义：

对于集合 “list = {“a”,”b”,”c”}”，使用方法 reverse(list) 后，集合将变成 {"c","b","a"}
