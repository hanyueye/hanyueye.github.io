Map 定义

Map 是一种键值对的形式： Map<K, V>，键不能重复，且一个键对应一个值。K 与 V 分别代表泛型的 Key 与 Value

Map 集合遍历

获取 Map 集合中所有的键值对有两种方式，就是 entrySet 与 keySet。

keySet

keySet() 其实就是获取所有的 Key 存储到了 Set 集合中，只要用 Set 的迭代器，遍历 Set 集合，就可得到所有的 Key 值，然后根据 Key 值即可获取到对应的 Value 值。

entrySet

entrySet() 则是得到了 Map 中的映射关系，再根据其映射关系，直接 getKey 和 getValue 即可获取到对应的值。

常见子类

  • Hashtable：底层是哈希表数据结构，不可存入 null 键 null 值，线程同步，效率较低。

  • HashMap：底层是哈希表数据结构，允许 null 键 null 值，但不推荐使用，线程不同步，效率高

  • TreeMap：底层是二叉树数据结构，线程不同步，可以用于给 Map 集合中所有的键进行排序。

HashMap 与 TreeMap 在使用上类似于 HashSet 与 TreeSet

使用 HashMap 存储自定义数据类型，要重写 hashCode 与 equals 方法

使用 TreeMap 存储，要让元素具备比较性。可以让元素本身具备比较性或者创建一个比较器
